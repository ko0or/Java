기존에 자료형이 ① 일반자료형 ② 객체자료형 의 존재만 알았었는데
이번 제네릭을 공부하며 놀랐던점

[1] String이 객체자료형이었어 ..? 
    int, String 모두 일반자료형인줄 알았는데
    String은 객체자료형이었다.. 
    (그러고보니 얘만 첫글자가 대문자였는데, 이유가 있었네)

[2] Object 카멜레온 썰
    그외에도 Object 라는 객체자료형도 알게되었다
    할당된 값에따라 자료형이 바뀌는 카멜레온 같은 존재다(?)

    숫자를 넣고 .getClass().getName() 을 해보면
    java.lang.Integer 가 나온다.
    하지만, 그렇다고 산술연산이 되는건 아니더라
    ( etc.java 파일에서 테스트 해봤었다 )

[3] 제네릭
    제네릭은 Object와 비슷한 느낌이다.
    왜냐면, 자료형이 미리 정해져있는게 아니기때문인데

    대신 자료형만 바뀌는게 아니라 그 성질(?)까지 바뀌는게 특징이다

[4] 제네릭 사용법
    파이썬의 딕셔너리 같은 느낌으로 이해했다 ( 키값 : 벨류값 )



class 제네릭클래스<암거나> {
    암거나 info;
    제네릭클래스(암거나 info) { this.info = info }
}

main에서는  인스턴스 할당해서 쓰면된다
위와같이 생성자를 통한 방식으로 선언과 동시 초기화를 하든
아님 게터 세터를통해 할당을 하든 편한대로 쓰면된다
